# RedBookAgent V1 双人开发功能负责书

更新时间：2026-04-21

## 1. 目标

V1 目标不是做自动爬取、自动发布或绕过平台风控，而是把“小红书半自动运营闭环”打磨到可试运营：

```text
赛道配置 -> 热点录入/导入 -> AI 选题分析 -> AI 二创草稿 -> 人工审核 -> 发布排期 -> 数据回收 -> 复盘分析 -> 下一轮选题建议
```

当前项目已经具备基础 CRUD、菜单权限、工作台、发布日历、复盘看板、AI 降级流程和基础闭环动作。接下来两人开发重点放在“V1 未完成和未产品化的功能”。

## 2. 当前已具备能力

- 后端已有 `redbook-agent` 模块，包含账号、赛道、人设、产品、提示词、敏感词、热点、热点分析、草稿、发布计划、数据回收、复盘报告等基础接口。
- 前端已有“小红书运营”菜单、通用 CRUD 页面、运营工作台、发布日历、复盘看板。
- 已支持热点 AI 分析、分析生成草稿、草稿加入排期、发布计划标记已发布、生成数据回收记录、生成复盘报告、复盘建议回流热点池。
- 已支持 Dify / FastGPT 配置接入，未配置时使用本地模板降级。
- 已支持通用 Excel 导入导出。

## 5. 推荐并行开发顺序

### 第 1 周：补齐闭环硬缺口

A：

- 补草稿审核通过/退回接口。
- 补状态流转校验。
- 补草稿版本表和版本 API。
- 统一数据回收指标计算。

B：

- 把 RedBook 通用 CRUD 里的 ID 输入改成选择器。
- 把状态输入改成下拉和 Tag。
- 做草稿审核弹窗和发布文案复制体验。
- 在现有工作台上补待办跳转和状态筛选。

第 1 周验收：

- 一个热点可以完成“分析 -> 草稿 -> 审核通过/退回 -> 加入排期”。
- 运营人员不需要手动复制 ID。
- 草稿每次编辑能留下版本记录。

### 第 2 周：补运营效率

A：

- 热点导入预校验和重复检查。
- 发布计划延期/取消/补录链接接口。
- 数据回收节点完整性校验。
- AI 输出 JSON 校验和错误记录。

B：

- 热点详情、分析详情、草稿版本对比。
- 发布日历内编辑、延期/取消、补录链接。
- 数据回收批量录入页面。
- 导入错误提示和模板下载入口。

第 2 周验收：

- 50 条热点可以批量导入，错误行有提示。
- 20 条草稿可以排期，并在日历完成调整。
- 发布后可以按 2h/24h/72h/7d 录入数据。

### 第 3 周：补复盘和试运营

A：

- 复盘报告按时间/赛道/账号生成。
- 权限按钮 SQL。
- 初始化样例数据和部署说明。
- 后端接口测试。

B：

- 复盘看板筛选和报告生成向导。
- 复盘建议回流热点池的前端确认流程。
- 前端关键流程自测。
- V1 操作说明。

第 3 周验收：

- 能从已发布和已回收数据生成复盘报告。
- 能把下一轮选题建议回流热点池。
- 能用样例数据演示完整 V1 流程。

## 6. 接口协作约定

- A 新增或修改接口时，同步写清楚 URL、方法、请求体、返回体、状态含义。
- B 新增页面时，先确认是否需要后端分页、筛选、批量接口，避免一次性拉全量数据。
- 所有状态值统一使用英文枚举，页面显示中文标签。
- 所有关系字段数据库仍保存 ID，页面显示名称；导出时尽量包含 ID 和名称两列。
- AI 接口必须允许失败降级，不能因为远程 AI 不可用阻断 V1 主流程。
- 导入接口必须返回可读错误信息，至少包含行号、字段和原因。

## 7. V1 不进入本次开发的范围

- 自动爬取小红书内容。
- 模拟登录小红书。
- 自动发布小红书笔记。
- 自动评论、自动私信、批量养号。
- 绕过平台风控或未授权采集、存储、搬运平台内容。

## 8. 最终验收清单

- 可以管理账号、赛道、人设、产品卖点、提示词、敏感词。
- 可以批量导入 50 条热点，并筛出可分析热点。
- 可以 AI 生成 30 条选题分析。
- 可以人工筛选并生成 20 条草稿。
- 可以审核通过、退回、编辑草稿，并保留版本。
- 可以将通过草稿加入发布计划，并在日历里调整。
- 可以复制发布文案，人工发布后补录链接并标记已发布。
- 可以录入或导入 2h/24h/72h/7d 数据。
- 可以查看工作台、发布日历、复盘看板。
- 可以生成 AI 复盘报告，并把下一轮建议回流热点池。
- 权限、导入导出、异常提示、部署说明齐全。
