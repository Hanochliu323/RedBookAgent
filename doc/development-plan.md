# 小红书运营智能体 V1 开发计划

## 1. 项目定位

本项目 V1 定位为“小红书半自动运营闭环系统”，用于支持固定赛道的热点整理、AI 二创、人审排期、数据回收和复盘分析。

V1 不做自动爬取小红书、模拟登录、自动批量发布、自动评论私信等高风险能力，而是先做稳定、合规、可持续迭代的运营 Copilot。

核心闭环：

```text
赛道配置 -> 热点录入/导入 -> AI 选题分析 -> AI 二创草稿 -> 人工审核 -> 发布排期 -> 数据回收 -> 复盘分析 -> 下一轮选题建议
```

## 2. 技术选型

### 2.1 后台底座

- JeecgBoot
- Java / Spring Boot
- Vue / Ant Design Vue
- MyBatis-Plus

选择原因：

- 适合快速搭建后台管理系统
- 自带用户、角色、权限、菜单、字典、日志等基础能力
- 低代码能力适合快速生成赛道、热点、草稿、数据表单页面
- 支持 Excel 导入导出，适合 V1 的半自动数据流

### 2.2 AI 能力

优先支持：

- Dify
- FastGPT

后续可扩展：

- OpenAI API
- 通义千问
- 智谱
- DeepSeek
- 本地大模型服务

AI 能力主要负责：

- 热点拆解
- 选题评分
- 标题生成
- 小红书笔记正文生成
- 封面文案生成
- 标签推荐
- 内容风险提示
- 数据复盘分析
- 下一轮选题建议

### 2.3 数据库

- MySQL

主要存储：

- 账号信息
- 赛道配置
- 人设配置
- 产品卖点
- 热点内容
- AI 分析结果
- 笔记草稿
- 发布计划
- 运营数据
- 复盘报告
- 提示词模板

### 2.4 数据导入导出

- Excel 导入
- Excel 导出

V1 典型使用场景：

- 批量导入热点标题和链接
- 批量导入发布后数据
- 导出草稿内容给运营人员审核

## 9. V1 验收标准

V1 上线后，需要能完整跑通以下流程：

```text
录入 50 条热点
  -> AI 生成 30 条选题分析
  -> 人工筛选 20 条草稿
  -> 排期发布
  -> 回收 2h/24h/72h/7d 数据
  -> 生成复盘报告
  -> 推荐下一批选题
```

功能验收：

- 可以管理账号、赛道、人设、产品卖点
- 可以批量导入热点
- 可以对热点进行 AI 分析
- 可以生成小红书笔记草稿
- 可以人工审核和修改草稿
- 可以生成发布计划
- 可以录入和导入发布后数据
- 可以查看基础数据看板
- 可以生成 AI 复盘报告

业务验收：

- 能判断哪些赛道值得继续做
- 能判断哪些内容类型表现更好
- 能沉淀高表现标题和封面文案
- 能发现低表现内容的原因
- 能给下一轮选题提供明确建议

## 12. 关键风险和处理策略

### 12.1 平台规则风险

风险：

- 自动爬取、自动登录、自动发布可能触发平台规则和账号风险。

策略：

- V1 坚持半自动模式
- 使用人工录入和 Excel 导入
- 发布保留人工确认
- 数据回收以人工录入和导入为主

### 12.2 内容同质化风险

风险：

- AI 生成内容容易变成模板化表达。

策略：

- 强制绑定账号人设
- 强制输出原创化角度
- 保留人工审核
- 建立高表现内容库和禁用表达库

### 12.3 数据质量风险

风险：

- 手动录入数据可能不完整或不准确。

策略：

- Excel 模板标准化
- 数据必填项校验
- 采集节点统一
- 数据异常提醒

### 12.4 AI 输出不稳定风险

风险：

- AI 输出格式不稳定，影响系统解析。

策略：

- 要求 AI 返回 JSON
- 增加输出格式校验
- 失败后允许重新生成
- 保留人工编辑入口

## 13. 下一步行动

建议下一步按以下顺序推进：

1. 初始化 JeecgBoot 项目。
2. 建立 MySQL 数据库。
3. 先实现基础配置、热点池、草稿三个核心模块。
4. 选择 Dify 或 FastGPT 作为第一个 AI 工作流平台。
5. 跑通“热点导入 -> AI 分析 -> 草稿生成”的最小链路。
6. 再补发布计划、数据回收和复盘看板。
